# GLoRS (Global Long-term Reservoir Sedimentation) - Source Code

This repository contains the source code and computational pipeline used to generate the **GLoRS** dataset. The pipeline leverages the **RECLAIM** framework to predict annual reservoir sedimentation and capacity loss globally.

## Repository Structure

- **`paper_global_reclaim_application_v1.ipynb`**: The primary research notebook used to generate the GLoRS v1.0 dataset. It includes:
  - Data ingestion from multiple global sources (GRILSS, RAT 3.0, GLC-SHARE, HWSD).
  - Feature engineering and transformation (log-scaling, area calculations).
  - Application of the RECLAIM machine learning ensemble.
  - Export of time-series and aggregated results.
- **`apply_reclaim.ipynb`**: Implementation of the RECLAIM model application logic for specific reservoir subsets and experimental validation.

## Prerequisites

To run these notebooks, the following frameworks and libraries are required:

1. **RECLAIM Framework**: The core machine learning logic for capacity loss prediction.
2. **RAT 3.0 (Reservoir Assessment Tool)**: For satellite-derived hydrological time-series.
3. **Python Environment**:
   - `geopandas`
   - `pandas`
   - `numpy`
   - `catboost` / `xgboost` / `lightgbm` (Ensemble components)
   - `scikit-learn`
   - `matplotlib` / `seaborn`

## Usage

1. Ensure the **RECLAIM** repository is available in your Python path.
2. Place the required input data (catchment boundaries, GRILSS metadata) in the expected directory structure.
3. Run `paper_global_reclaim_application_v1.ipynb` to reproduce the global dataset.
